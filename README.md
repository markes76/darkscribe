<h1 align="center">Darkscribe</h1>

<p align="center">
  Real-time call transcription and note-taking that lives in your Obsidian vault.
</p>

<p align="center">
  macOS only (Apple Silicon) &bull; Electron + React + TypeScript &bull; 100% local data
</p>

---

Darkscribe captures audio from calls (Zoom, Teams, Google Meet, phone) and in-person conversations, transcribes in real-time using the OpenAI Realtime API, generates structured summaries with sentiment analysis, and saves everything as markdown in your Obsidian vault.

It also surfaces relevant context from your past notes during live calls, so you always have history at your fingertips.

## Features

### Live Transcription
- **Dual-channel audio capture** — mic (local speakers) + system audio (remote participants via Zoom/Teams/etc.) captured simultaneously using ScreenCaptureKit
- **Real-time streaming transcription** via OpenAI Realtime API (two parallel WebSocket sessions)
- **Multi-language support** — auto-detect or set preferred languages (up to 3). For multilingual users, Whisper auto-detects per sentence to prevent language locking
- **Language indicator badges** — each transcript segment shows a detected language tag (EN, HE, etc.)
- **Anti-hallucination filters** — post-processing pipeline discards silence hallucinations, duplicate segments (>90% overlap), and wrong-language output
- **Smart VAD** — different voice activity detection settings for mic vs system audio, with periodic forced commits for continuous speech
- **Vocabulary hints** — loads your Notetaker Skill file before each session so the transcription knows your preferred names, acronyms, and terms

### Audio-Based Summary Generation (Background Processing)
After a call ends, Darkscribe runs a background processing pipeline for significantly higher-quality output:

1. **Whisper API re-transcription** — sends the WAV recording to OpenAI's Whisper API for a more accurate transcript than the live Realtime API provides
2. **GPT-4o summary** — generates the structured summary from the Whisper transcript (not the live draft)
3. **Gemini voice insights** (optional) — if a Gemini API key is configured, analyzes voice tone, speaker dynamics, and emotional shifts directly from the audio
4. **Automatic vault update** — overwrites the initial live-transcript-based vault notes with the improved versions

The pipeline runs asynchronously and non-blocking. You can start a new call, navigate the app, or even close it while processing runs. Status is shown on session cards (amber spinner = analyzing, green check = complete).

**Large file handling:** WAV files over 25MB are automatically split into 10-minute chunks, sent to Whisper in parallel, and stitched back together with correct timestamps.

### Post-Call Summary
- **Structured summaries** generated by GPT-4o: overview, key topics, action items (with owners), decisions, follow-ups
- **Detailed sentiment analysis**: overall tone, emotional arc, participant dynamics, per-topic sentiment, key moments with cited evidence, concerns, risks, and a follow-up recommendation
- **Live vs. Final transcript toggle** — switch between the real-time draft and the improved Whisper version
- **Shareable meeting summary** — professional format for sending to attendees, with detected participants and action item table

### In-App Audio Player
- **Waveform visualization** — generated from the WAV recording data
- **Full playback controls** — play/pause, clickable seek bar, current time / total duration
- **Playback speed** — 0.75x, 1x, 1.25x, 1.5x, 2x
- **Volume control**
- Available on the post-call summary screen and when reviewing past sessions
- Shows "Recording deleted" gracefully when audio has been removed

### Storage Management
- **Per-session audio info** — file size displayed on session cards
- **Delete individual recordings** — confirmation dialog, frees disk space while preserving transcript and summary
- **Bulk management** in Settings > Storage:
  - Total disk usage display
  - Delete recordings older than 7/30/90 days
  - Delete all recordings
- **Auto-delete** — configurable retention period (never, 7, 30, or 90 days), runs on app launch

### Second Brain Mode (Voice Notes)
- Solo recording mode for brainstorming, journaling, or thinking out loud
- Generates "thought summaries" instead of meeting summaries: key ideas, questions to explore, connections, raw gems
- Saves to configurable Notes/ subfolders (Ideas, Research, Brainstorm, Journal)

### Session Persistence (Never Lose Data)
- **Auto-save to disk** — transcripts save every 10 seconds during recording. Summaries, web searches, and references persist immediately on generation
- **Crash recovery** — if the app crashes mid-recording, restart and find your partial transcript waiting
- **Review mode** — reopen any past session from the home screen. Full summary loads from disk without re-calling the API
- **Session status** — color-coded badges: green (saved to vault), blue (summarized), red (interrupted), amber (processing)
- **Auto-save to Obsidian** — optional toggle in Settings. When enabled, transcripts and summaries save to your vault automatically after generation
- **Auto-update vault after processing** — when background analysis completes, vault notes are automatically overwritten with the improved version (configurable)
- **Note references** — attach Obsidian notes as context before or during a call. Reference content enriches the GPT-4o summary prompt. Saved as [[wikilinks]] in frontmatter
- **Navigation-safe recording** — navigate to Settings or Home during a recording without losing the live transcript. A "Recording in progress" banner lets you return instantly

### Obsidian Vault Integration
- All notes saved as markdown with YAML frontmatter — transcripts, summaries, voice notes, web search references
- **Complete vault notes** — each saved note includes: overview, key topics, action items, decisions, follow-ups, full sentiment analysis, and the complete transcript with speaker labels and timestamps
- **Version tracking** — vault frontmatter includes `transcript_version` (live/final), `processed_with`, and `processed_at` fields
- **Context surfacing** — during live calls, Darkscribe automatically extracts entities from the conversation and searches your vault for related past notes, surfacing them in a side panel with "Open in Obsidian" links
- **Sticky context cards** — surfaced notes stay visible until manually dismissed
- **Vault search** — manual search across your Obsidian vault during calls
- **Web search** — Tavily + OpenAI fallback, with "Add to Summary" to include web results in your call summary
- **Obsidian deep links** — "Open in Obsidian" buttons on context cards, saved notes, and session list entries

### Self-Teaching Notetaker
- A persistent **Notetaker Skill file** (`System/Notetaker Skill.md`) that Darkscribe reads before every operation
- Vocabulary corrections, formatting preferences, and summarization patterns accumulate over time
- When you edit a summary, Darkscribe diffs the changes and logs learnings back to the Skill file
- The more you use it, the better it gets

### Obsidian Console Design System
- **Dark mode first** — deep charcoal, obsidian backgrounds with amber/gold accents
- **Light mode** — warm mineral paper tones with graphite text
- **Frosted glass surfaces** — backdrop-filter blur, layered depth
- **Premium typography** — Outfit (display), Inter (body), JetBrains Mono (timestamps)
- **Recording animations** — breathing pulse, amber glow on active recording

## Prerequisites

### Required
- **macOS 13.0+** (Ventura or later), Apple Silicon (arm64)
- **Node.js 18+** and npm
- **Xcode Command Line Tools** (for building the Swift audio capture binary)
- **OpenAI API key** — used for live transcription (Realtime API), Whisper re-transcription, and GPT-4o summarization. This is the core dependency. [Get an API key](https://platform.openai.com/api-keys)
- **Obsidian** with the [Local REST API](https://github.com/coddingtonbear/obsidian-local-rest-api) community plugin installed and enabled

### Optional
- **Tavily API key** — enables web search during calls. Without it, web search falls back to OpenAI or is unavailable. [Get a Tavily key](https://tavily.com/)
- **Gemini API key** — enables voice tone and speaker dynamics analysis from audio recordings (powered by Google's Gemini). Without it, sentiment analysis still works via GPT-4o text analysis, but voice-specific insights (tone, energy, emotional shifts from audio) are skipped. [Get a Gemini key](https://aistudio.google.com/apikey)

| **Obsidian REST API** | Yes | Reading/writing vault notes (local, no cost) | Free |
| **Tavily** | No | Web search during calls | Free tier available |

**Estimated cost per 30-minute call:** ~$0.20 (Whisper + GPT-4o). With Gemini: ~$0.25.

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/markes76/darkscribe.git
cd darkscribe
```

### 2. Install dependencies

```bash
npm install
```

### 3. Build the Swift audio capture binary

```bash
npm run build:swift
```

This compiles the ScreenCaptureKit-based audio capture tool and places it at `resources/AudioCapture`.

### 4. Set up the Obsidian Local REST API plugin

1. Open Obsidian with your vault
2. Go to **Settings** (gear icon) > **Community plugins**
3. If community plugins are disabled, click **Turn on community plugins**
4. Click **Browse** and search for **"Local REST API"**
5. Click **Install**, then **Enable**
6. Go back to **Settings** > scroll down to **Local REST API** in the left sidebar
7. Toggle ON **"Enable Non-Encrypted (HTTP) Server"**
8. Copy the **API Key** shown at the top
9. Note the **port number** (usually 27123 or 27124)

### 5. Run in development mode

```bash
npm run dev
```

The app will launch and walk you through onboarding (API keys, permissions, vault connection, folder selection).

### 6. Build for production (optional)

```bash
npm run package
```

This creates a signed `.app` bundle in `dist/mac-arm64/`. Copy it to `/Applications/` to install.

## macOS Permissions

Darkscribe requires two macOS permissions:

- **Microphone** — for capturing your voice. Granted via the standard macOS permission dialog.
- **Screen Recording** — for capturing system audio (remote callers via Zoom/Teams/etc.) using ScreenCaptureKit. Grant this in **System Settings > Privacy & Security > Screen Recording**.

After granting Screen Recording permission, you must **restart the app** for it to take effect.

### What Gets Saved to Vault

Each call generates two notes in the vault:

**Summary note** (`Calls/Summaries/`):
- YAML frontmatter: date, title, participants, duration, recording_status, transcript_version, processed_with
- Overview
- Key Topics (as bullet list)
- Action Items (as checkboxes with owners)
- Decisions
- Follow-ups
- Full Sentiment Analysis (tone, arc, dynamics, per-topic sentiment, key moments, signals, concerns, risks, recommendation)
- Voice Insights from Gemini (if configured)
- Web Searches (if any were added during the call)
- Full Transcript with timestamps and speaker labels (You/Them)

**Transcript note** (`Calls/Transcripts/`):
- YAML frontmatter: date, title, participants, duration, recording_status
- Full transcript with timestamps and speaker labels

## Architecture

Darkscribe is a three-process Electron application:

### Electron Main Process (`src/main/`)
- `audio-bridge.ts` — Spawns the Swift AudioCapture binary, accumulates raw PCM chunks, forwards to renderer via IPC
- `obsidian-api.ts` — Direct HTTP client for Obsidian's Local REST API (read, write, search, patch notes)
- `session-manager.ts` — Persists session data to `userData/sessions.json` with per-session file storage
- `keychain.ts` — Encrypts API keys using Electron's safeStorage (macOS Keychain-backed)
- `config.ts` — App configuration at `userData/config.json`
- `tavily-search.ts` — Web search via Tavily API with OpenAI fallback
- `ipc-handlers.ts` — IPC bridge for permissions, dialogs, shell operations, file I/O, storage management
- `recording-writer.ts` — WAV file writer (16kHz mono, downsampled from 24kHz capture) with retention-based auto-purge
- `whisper-transcriber.ts` — Whisper API client with automatic chunking for files >25MB
- `background-processor.ts` — Async pipeline: Whisper → GPT-4o → Gemini → vault sync

### Preload (`src/preload/`)
- `index.ts` — Typed IPC bridge exposing `window.darkscribe` API to the renderer

### Renderer (`src/renderer/src/`)
- `App.tsx` — State machine with navigation-safe recording (MainApp stays mounted via CSS during navigation)
- `services/openai-realtime.ts` — Dual-channel WebSocket transcription with anti-hallucination and language enforcement
- `services/summarizer.ts` — GPT-4o summary generation with sentiment analysis
- `services/context-surfacer.ts` — Entity extraction + vault search for context surfacing
- `hooks/useRealtimeTranscription.ts` — Coordinates audio capture, WebSocket sessions, transcript state, and post-processing filters
- `hooks/useContextSurfacing.ts` — Polls transcript for entities, searches vault, surfaces context cards
- `components/AudioPlayer.tsx` — Waveform player with speed/volume controls, loads WAV via IPC Blob URL

### Swift AudioCapture (`swift/AudioCapture/`)
- ScreenCaptureKit binary capturing system audio as raw Int16 PCM at 24kHz mono
- Outputs to stdout; Electron reads via child process pipe

## How It Works

### Audio Pipeline
1. **System audio**: Swift binary uses ScreenCaptureKit to capture all system audio (Zoom, YouTube, etc.) as 24kHz mono PCM
2. **Microphone**: Browser's `getUserMedia` captures mic audio in the renderer process
3. **Dual WebSocket sessions**: Two separate OpenAI Realtime API connections — one for mic, one for system audio — enabling independent transcription
4. **Post-processing filter**: Each transcript segment passes through language detection, deduplication, and minimum-length checks before display
5. **Recording**: Both channels are mixed and downsampled to 16kHz mono WAV, saved locally per session

### Background Processing Pipeline
1. Call ends → live transcript and quick GPT-4o summary generated immediately
2. Background: WAV file sent to Whisper API (chunked if >25MB) → accurate transcript
3. Background: Whisper transcript sent to GPT-4o → improved summary with sentiment
4. Background: If Gemini key configured, audio sent to Gemini → voice insights
5. Background: If vault was already saved, notes updated with improved versions
6. Toast notification when complete; session card shows green checkmark

### Context Surfacing
1. Every 30 seconds during a call, the last 60 seconds of transcript is sent to GPT-4o-mini
2. Named entities are extracted (people, companies, projects, topics)
3. Each entity is searched in the Obsidian vault via the REST API
4. Matching notes appear as sticky cards in the Context panel with excerpts and "Open in Obsidian" links
5. Vocabulary corrections from the Notetaker Skill file are applied to search queries

### Sentiment Analysis
After a call ends, GPT-4o analyzes the full transcript for:
- Overall emotional tone and how it evolved
- Per-topic sentiment breakdown with cited evidence
- Participant dynamics and engagement levels
- Implicit concerns detected from hedging language
- Positive signals and risk indicators
- An actionable recommendation for the next interaction

## Data Privacy

- **All data stays local.** Transcripts, summaries, and recordings are stored in your Obsidian vault and Electron's userData directory.
- **No cloud sync.** No telemetry. No analytics.
- **API calls** go to:
  - **OpenAI** — live transcription (streamed audio), Whisper re-transcription (WAV upload), GPT-4o summarization (text)
  - **Tavily** (optional) — web search queries
  - **Google Gemini** (optional) — audio analysis (WAV upload for voice insights)
- **Audio is not stored** by OpenAI or Google beyond the API session.
- **API keys** are encrypted using Electron's safeStorage (macOS Keychain-backed).
- **Audio files** are stored locally only. Never uploaded to Obsidian vault.

## Known Limitations

- **macOS only** — ScreenCaptureKit is an Apple framework. No Windows/Linux support.
- **Apple Silicon only** — The Swift binary is compiled for arm64.
- **Obsidian must be running** — The Local REST API plugin serves from within Obsidian. If Obsidian is closed, vault operations fail gracefully with a "Vault not connected" message.
- **Dual-channel timing** — Transcripts from mic and system audio may occasionally appear slightly out of order (1-3 seconds) due to independent WebSocket processing. The saved transcript sorts by timestamp.
- **System audio via Bluetooth** — ScreenCaptureKit captures system audio at low levels when routing through Bluetooth (AirPods). The silence threshold is set very low (5) to accommodate this.
- **Whisper API file limit** — 25MB per upload. Automatically handled by splitting recordings into 10-minute chunks.

# Cowork Integration (Optional)

Darkscribe pairs with [Claude Cowork](https://claude.ai) to automate daily vault maintenance, review, and insight surfacing. This is entirely optional but adds significant value once your vault has a few weeks of call transcripts and voice notes.

## What Cowork Does

Cowork is Anthropic's desktop agent that runs scheduled tasks on your local machine. When connected to your Obsidian vault via the same Local REST API that Darkscribe uses, it can autonomously review, clean, and organize your notes on a daily cadence.

Think of it as a second pass on everything Darkscribe captures. Darkscribe records and transcribes in real-time. Cowork reviews the output later, fixes errors, extracts patterns, and builds daily briefings.

## Setup

### Prerequisites

- [Claude Desktop](https://claude.ai/download) with a paid plan (Pro, Max, Team, or Enterprise)
- The same Obsidian Local REST API plugin that Darkscribe uses (already installed if Darkscribe is working)

### Create the Cowork Project

1. Open Claude Desktop > Cowork tab
2. Create a new Project called **"Darkscribe Daily"**
3. Set the working folder to your Obsidian vault path (e.g., `/path/to/your/vault`)
4. Connect the Obsidian MCP connector, or give Cowork access to the vault folder
5. Darkscribe auto-creates a `System/CLAUDE.md` file in your Darkscribe vault folder during onboarding. Cowork reads this as persistent project context.

### Scheduled Tasks

Set up these three scheduled tasks in Cowork > Scheduled > New Task. Adjust times and days to your work schedule.

---

### Morning Review

**Schedule:** Every work day, before your commute (e.g., 7:00 AM)

**What it does:**
- Reads the Notetaker Skill file to apply your learned preferences
- Reviews all call summaries and transcripts from the previous day
- Cleans transcripts: fixes transcription errors, applies vocabulary corrections, ensures consistent formatting
- Validates summaries: checks that every action item has an owner and deadline
- Reviews voice notes and cross-references them with calls from the same day
- Checks saved web searches in Resources/References/
- Creates a daily note with: yesterday's calls, voice notes, consolidated action items, pending follow-ups, and a blank "Today's Focus" section
- Updates the Notetaker Skill file with any new vocabulary corrections or patterns

**Prompt:**

```
Read {{VAULT}}/{{DARKSCRIBE_FOLDER}}/System/Notetaker Skill.md first. Apply its rules.

Review all files in {{DARKSCRIBE_FOLDER}}/Calls/Summaries/ and {{DARKSCRIBE_FOLDER}}/Calls/Transcripts/ with yesterday's date.

For each transcript: fix transcription errors using the Skill file vocabulary, clean formatting, write the cleaned version back.

For each summary: verify action items have owners and deadlines. If a summary is thin or missing sections, regenerate from the transcript.

Review any voice notes in {{DARKSCRIBE_FOLDER}}/Notes/ subfolders with yesterday's date. If a voice note connects to a call from the same day, add wikilink references between them.

Check {{DARKSCRIBE_FOLDER}}/Resources/References/ for web searches from yesterday.

Collect ALL action items from yesterday. Deduplicate.

Create {{DARKSCRIBE_FOLDER}}/Daily/YYYY-MM-DD.md (today) with:
## Yesterday's Calls
(one-line summary + [[wikilink]] each)
## Yesterday's Voice Notes
(same format, skip if none)
## Open Action Items
(consolidated, grouped by person)
## Pending Follow-ups
(due today or overdue)
## Today's Focus
(leave blank)

Update Skill file if you corrected recurring errors or noticed patterns.
```

Replace `{{VAULT}}` with your vault path and `{{DARKSCRIBE_FOLDER}}` with your Darkscribe folder (default: `Work/Darkscribe`).

---

### Weekly Rollup

**Schedule:** First day of your work week (e.g., Sunday 7:30 AM)

**What it does:**
- Reads all summaries, voice notes, daily notes, and web searches from the past 7 days
- Creates a weekly review note with: call/note counts, key themes, consolidated action items, stale items, cross-references between related notes, orphan checks, tag hygiene, and a Skill file self-assessment
- Appends the self-assessment to the Notetaker Skill file

**Prompt:**

```
Read {{DARKSCRIBE_FOLDER}}/System/Notetaker Skill.md first.

Read all files from the past 7 days in:
- {{DARKSCRIBE_FOLDER}}/Calls/Summaries/
- {{DARKSCRIBE_FOLDER}}/Notes/ (all subfolders)
- {{DARKSCRIBE_FOLDER}}/Daily/
- {{DARKSCRIBE_FOLDER}}/Resources/References/

Create {{DARKSCRIBE_FOLDER}}/Notes/Meetings/Weekly-Review-YYYY-MM-DD.md with:

## This Week at a Glance
Total calls, voice notes, people spoken with, web searches saved.

## Key Themes
Topics that recurred across calls and notes. Group and link to sources.

## All Open Action Items
Consolidated, deduplicated, with [[wikilinks]]. Group by person. Mark overdue items.

## Stale Items
Action items carried forward 3+ days without completion.

## Connections
Voice note topics that overlap with call discussions. Cross-reference with wikilinks.

## Orphan Check
Transcripts without matching summaries. Generate summaries for orphans.

## Tag Hygiene
Tags appearing only once (potential typos). Obvious duplicates to consolidate.

## Skill File Self-Assessment
Patterns emerging, suggestions for improvement.

Append the self-assessment to the Skill file under "Patterns and Learnings."
```

---

### End-of-Day Digest

**Schedule:** Every work day, before end of day (e.g., 4:30 PM)

**What it does:**
- Appends a quick pulse check to today's daily note: calls made, voice notes captured, web searches saved, new action items, and anything that looks unfinished

**Prompt:**

```
Read {{DARKSCRIBE_FOLDER}}/System/Notetaker Skill.md first.

Find today's daily note at {{DARKSCRIBE_FOLDER}}/Daily/YYYY-MM-DD.md.
If it doesn't exist, create it with the standard template first.

Append:

## End of Day
Calls today: X (list with [[wikilinks]])
Voice notes today: X (list with [[wikilinks]])
Web searches saved: X
New action items: X
Unfinished: (transcripts without summaries, notes without tags, action items without owners)

Keep under 150 words.
```

## How It Compounds

The Notetaker Skill file is the compounding asset. Every scheduled task reads it before acting and writes back what it learned after. Over weeks:

- Vocabulary corrections accumulate (names, acronyms, company-specific terms)
- Summary style aligns closer to your preferences
- Cross-references between calls and voice notes surface unexpected connections
- Stale action items get flagged before they're forgotten
- Weekly reviews show trends you wouldn't notice day-to-day

The more you use Darkscribe, the smarter the Cowork reviews become.
